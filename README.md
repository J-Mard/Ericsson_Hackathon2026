# Ericsson Hackathon 2026 — 5G/6G Subsea Welding Ops

Proof-of-concept for a 5G/6G edge-computing pitch: two simulated North Sea oil
rigs, each with an autonomous **underwater welding drone** tethered by fibre to
a surface **5G edge buoy**. Marine weather is replayed from an Open-Meteo CSV;
a scikit-learn decision tree drives mission state; a local Qwen 2.5 3B model
(optional) narrates state transitions into a chat log.

## Run

```bash
python -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt

# Optional — only if you want LLM narration. Template fallback works offline.
ollama pull qwen2.5:3b

streamlit run frontend/app.py
```

In the sidebar you can:

- Change tick interval (ms).
- Push the **Stress test** slider to multiply wave height (1.0–3.0×) on the
  normal data — a smooth way to ramp into `CAUTION` without triggering the
  full storm.
- Toggle LLM narration on/off.
- **Trigger Disaster** (top bar) — swaps the data source from
  `open-meteo-...csv` to [`storm.csv`](storm.csv) for the next 30 ticks. The
  storm arc plays from the start each time, building from 2.5 m waves up to
  a ~6.2 m peak at tick ~12 before subsiding. The rig card's "data:" badge
  flips to `storm.csv` while active so the audience sees the swap.

## Architecture

```
normal.csv ─┐
            ├──► data.py tick loop ──► per-station Tick
 storm.csv ─┘          ▲
                 disaster button      (selects which CSV feeds this tick)
                 stress slider        (multiplies wave/current on top)
                                   ▼
                      ml.py  (DecisionTree + λ, R)
                                   │
                   state change?   ├── yes ──► llm.py (1 sentence narration)
                                   │
                                   ▼
                           frontend/app.py Streamlit UI
                (KPI cards, Plotly charts, chat log, 5G latency meter)
```

**Two-dataset replay model.** The app never "fakes" weather with hard-coded
constants. Normal ticks pull from the real Open-Meteo CSV, and disaster
ticks pull from a curated 30-row storm profile. Switching data sources is
cleaner than overriding values and makes the demo honest: everything the
drones react to comes from a CSV the jury can open.

## ML explained

This section is what the jury cares about — all the "intelligence" lives in
[`core/ml.py`](core/ml.py) and has two parts: a reliability curve and a
decision tree.

### 1. Reliability curve

The pitch hook is the classic exponential reliability model:

```
R(t) = e^(-λt)
```

λ is assembled from three linear contributors, each with a hand-tuned
coefficient chosen so that calm sea + healthy weld gives a slow decay and a
storm gives a fast one:

```
λ = 0.05 · H_s        (wave height, metres)
  + 0.03 · U_c        (current velocity, m/s)
  + 0.02 · (100 − W)/100   (weld-integrity deficit, 0..1)
```

The sidebar renders the two formulas live with `st.latex`. The KPI cards show
`R` at a fixed 10-second horizon so NORMAL looks healthy (~0.5–0.6) while a
storm collapses R to near zero.

Illustrative values (`t = 10 s`):

| Scenario | H_s | U_c | W | λ | R(10s) |
| --- | --- | --- | --- | --- | --- |
| Calm sea, healthy weld | 0.5 | 0.3 | 95 | 0.035 | 0.705 |
| Moderate sea | 1.8 | 0.7 | 80 | 0.115 | 0.317 |
| Storm, weak weld | 4.5 | 1.8 | 35 | 0.293 | 0.054 |

### 2. Decision tree classifier

We train a shallow `DecisionTreeClassifier(max_depth=5)` on a synthetic
dataset of 3000 uniformly-sampled rows. Labels are generated by transparent
rules before training:

```python
if wave >= 3.0 or current >= 1.0 or weld < 40:
    label = "ABORT"
elif wave >= 1.8 or current >= 0.6 or weld < 65 or period < 5.0:
    label = "CAUTION"
else:
    label = "NORMAL"
```

These thresholds come straight from the brief (`wave > 3 m` or
`current > 1 m/s` ⇒ emergency retreat). The tree learns those rules back,
which is deliberate:

- **Interpretable**: the sidebar shows `feature_importances_` as a bar chart.
- **Fast**: sub-millisecond inference per tick, which supports the "Edge AI at
  1–2 ms" narrative.
- **Trustworthy on stage**: the predictions match an expert heuristic, so the
  jury can spot-check them.

### 3. Weld-integrity simulator

The marine CSV has no weld-quality signal, so `next_weld_integrity()` produces
a random walk biased downward by wave height:

```
weld_{t+1} = clip(weld_t − 0.4·H_s + N(0, 1.5²), 0, 100)
```

This is what causes weld quality to visibly decay during the disaster window,
feeding back into both λ (lower R) and the classifier (eventually flips state
to `ABORT`).

## Pitch hooks already baked in

- Live LaTeX of `R(t)` and `λ` in the sidebar.
- Decision-tree feature importances as a bar chart — "real ML, not vibes".
- 5G URLLC latency meter at 1–2 ms compared to a 40–60 ms 4G baseline.
- Disaster button for a live on/off demo.
- Stress slider for a smooth ramp into `CAUTION` → `ABORT`.

## Non-goals

- No live Open-Meteo API call (CSV replay only — deterministic demo).
- No parallel LLM inference (sequential, and only on state changes).
- No persistence — everything lives in `st.session_state`.
